## Satya Nadella：Agent 时代的操作系统——Microsoft Build 2026

> [原文连接](https://blogs.microsoft.com/blog/) 2026年5月（Microsoft Build 2026）

------

> 微软 CEO Satya Nadella 在 Build 2026 开发者大会上发表主题演讲，正式宣告"Agent 时代"的到来。在 Copilot 系列产品已经覆盖超过 10 亿用户之后，他提出了一个核心论断：**AI Agent 是继 PC 操作系统、移动操作系统之后，第三次"操作系统级别"的平台迁移。**

------

Satya Nadella 站在台上说了一句让所有开发者都停下来的话：

> "每一个应用，都将被一个 Agent 重写。问题不是'会不会'，而是'谁来写'。"

## 三次操作系统革命

他用历史做了类比：

**第一次**：1980 年代，DOS/Windows 的出现，让个人电脑成为生产工具。微软通过 OS 赢得了第一个三十年。

**第二次**：2007 年，iPhone/Android 的出现，让移动设备成为生活中心。微软错过了这次机会。

**第三次**：2024-2026 年，AI Agent 框架的出现，让"会思考、会行动的软件"成为可能。Nadella 的判断：**微软不会再错过这次。**

> "Copilot 不是一个功能，不是一个应用，它是一个新的操作系统层。它坐在你的所有应用之上，理解你的意图，调度你所有的工具。"

## 什么是 Agent 时代的核心变化

Nadella 给出了他对 Agent 时代的本质定义：

**旧范式（Application Era）**：
- 人类学习如何使用软件；
- 界面是固定的（按钮、菜单、表单）；
- 每个应用独立运作，数据孤岛。

**新范式（Agent Era）**：
- 软件学习如何理解人类意图；
- 界面是对话式的，或完全消失；
- Agent 可以跨应用、跨数据源完成任务，数据流动。

他举了一个具体例子：
> "过去，你要完成一份季度销售报告，需要打开 Excel 拉数据、打开 PowerPoint 做图表、发邮件给团队、在会议里讲解。未来，你只需要说一句话：'帮我准备下周的季度销售汇报'，Agent 完成所有步骤。"

## Copilot Studio：让每个企业构建自己的 Agent

这是 Build 2026 最重要的产品发布：**Copilot Studio 企业版**。

核心能力：
- 企业可以用自己的内部数据训练专属 Agent；
- Agent 可以访问企业所有 SaaS 工具（Salesforce、SAP、ServiceNow 等）；
- 内置权限管控和审计日志，满足企业合规需求。

Nadella 的判断：
> "企业软件的下一代形态，不是更好的 ERP，不是更好的 CRM，而是为每个工作流定制的 Agent。"

这对 Salesforce、ServiceNow 这些传统 SaaS 公司构成了直接威胁，也为新一代垂直 SaaS 创业公司打开了窗口。

## 开发者如何定位

Nadella 对开发者说了一段直接的话：

> "你们面前有两个选择：第一，继续构建传统应用，在五年内看着它被 Agent 取代；第二，现在就开始构建 Agent-first 的产品，成为这次迁移的受益者，而不是牺牲品。"

他给出了具体的判断框架——一个好的 Agent 产品需要三个条件：
1. **清晰的任务边界**：Agent 要解决什么具体问题，要有明确的定义；
2. **可靠的工具调用**：Agent 需要能稳定地调用外部系统，而不是只会聊天；
3. **人类在回路中**：对于高风险决策，Agent 应该暂停并请求人类确认，而不是自动执行。

## 关于 AI 的民主化

Nadella 始终坚持一个价值观：**AI 的价值，来自于让尽可能多的人获得 AI 能力，而不是让少数人获得超级 AI。**

他援引了一个数字：使用 GitHub Copilot 的开发者，平均编程效率提升 55%。这意味着一个之前只能维护系统的中级工程师，现在可以完成之前需要高级工程师才能完成的任务。

> **"AI 不是精英的工具，AI 是让每个人都有机会做精英才能做的事情的工具。这才是我们做这件事的意义。"**

**_"每一次技术平台的迁移，都会产生一批新的世界级公司。PC 时代有微软、Adobe、Oracle。移动时代有 Uber、Airbnb、抖音。Agent 时代的赢家，很多还没有诞生。"_**
